The Problem

Vision-only agents are slow & expensive

Most AI desktop agents screenshot → send to LLM → get coordinates → click. Repeat 10-20 times per task. There's a better way.

❌ Pure Vision Agents

  1. Screenshot the entire screen
  2. Send to GPT-4o/Claude ($$)
  3. Get back pixel coordinates
  4. Click at (x, y) — hope it's right
  5. Screenshot again... repeat 10x
~12s
Per task
~15,000
Tokens used

✅ Clawd Cursor

  1. Parse task into subtasks (1 text call)
  2. Query UI Automation tree
  3. Find element by name — click directly
  4. No screenshots needed for 80% of tasks
  5. LLM vision only as fallback
~2s
Per task
~500
Tokens used
Architecture

Parse → Accessibility → Action

A three-stage pipeline that tries the fastest approach first, falling back to vision only when needed.

1

Decompose Task

A single text-only LLM call breaks your task into atomic subtasks.

"Open Chrome and go to GitHub" →
["open Chrome", "go to github.com"]

⚡ 200ms
→
2

Action Router

Queries the Windows UI Automation tree — the same API screen readers (JAWS, NVDA) use. Finds elements by name, not coordinates.

Handles: open apps, type text, click buttons, navigate URLs, focus windows

✅ No LLM — 300ms
→
3

LLM Vision Fallback

Only when the router can't handle it: resized screenshot → vision LLM → coordinates → execute.

Used for complex/unknown UI only (~20% of tasks)

⚠️ Fallback — 2s
Features

Built for speed, safety, and reliability

🔍

UI Automation Tree

Queries the Windows accessibility API to find buttons, menus, and inputs by name — not by guessing pixel coordinates.

⚡

Zero-LLM Fast Path

Common tasks (open apps, type text, click elements) execute in milliseconds with zero AI calls.

🧠

Smart Decomposition

One cheap text-only LLM call breaks complex tasks into atomic steps the router can handle.

📸

Resized Screenshots

When vision is needed, screenshots are resized to minimize token usage while preserving accuracy.

🛡️

Safety Tiers

Three-tier confirmation system: auto for safe actions, preview for typing, confirm for destructive ops.

🔌

VNC Protocol

Connects as a standard VNC client. Watch the AI work in real-time from your own VNC viewer.

🛡️ Safety by Design

Every action is classified before execution. Destructive actions always require human approval.

🟢 Auto

Open apps, navigate, read screen, switch windows. Executes immediately.

🟡 Preview

Type text, fill forms. Logged before executing so you can see what's happening.

🔴 Confirm

Send messages, delete files, purchases. Pauses and waits for your explicit approval.

Get Started

Up and running in 2 minutes

Terminal 
# Clone and build
git clone https://github.com/AmrDab/clawd-cursor.git
cd clawd-cursor
npm install
npm run build

# Configure your AI API key
cp .env.example .env
# Edit .env → AI_API_KEY=sk-...

# Start (make sure VNC server is running)
npm start -- --vnc-password yourpass

# Send a task
curl http://localhost:3847/task \
  -d '{"task": "Open Chrome and go to github.com"}'
Prerequisites: Node.js 20+ • VNC Server (TightVNC, UltraVNC, etc.) • PowerShell (Windows) • AI API key (Anthropic or OpenAI)
